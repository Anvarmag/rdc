# Скрипты (Vanilla JS)

В этой папке хранится вся интерактивная клиентская логика проекта. 
**Внимание: Проект не использует Node.js или фронтенд-фреймворки (типа React).** Весь код написан на "ванильном" ES6+ JavaScript.

## Структура папки

- **`main.js`** — Точка входа приложения. Оркестратор модулей: инициализирует хэдер, подгружает и рендерит данные из `data/`.
- **`config.js`** — (Не используется в статик-релизе, заглушка) Файл глобальной конфигурации.
- **`cookie-consent.js`** — Логика отображения баннера об использовании куки-файлов.

### 📁 `modules/`
Изолированные функциональные блоки (IIFE/модули), отвечающие за отдельные зоны ответственности:
- **`header.js`** — Логика адаптивной шапки (сжатие при скролле, мобильное бургер-меню, плавная прокрутка по якорям без изменения URL).
- **`team.js`** — Логика рендеринга карточек врачей и управление модальными окнами профилей специалистов.

### 📁 `data/`
Данные для отрисовки на клиенте.
- **`doctors.js`** — Большой массив объектов (JSON-подобная структура), содержащий профили, регалии, опыт и переводы регалий для всех специалистов РДЦ.
- **`services.js`** — Массив объектов с описанием услуг (МРТ, КТ и т.д).

### 📁 `i18n/`
- **`translations.js`** — Глобальный словарь переводов (RU, TT, EN). Важно: он парсится статическим скриптом генерации `build.py` (на уровень выше, в корне) и встраивается непосредственно в HTML. Сам файл остается здесь только как источник данных (база).
