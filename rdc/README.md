## О проекте

**РДЦ (Радиологический дата-центр)** — это лендинг для телемедицинского сервиса, предоставляющего услуги дистанционного описания медицинских изображений.

Сайт был оптимизирован для сверхбыстрой работы:
- Полностью отказались от `Node.js`, сборщиков и тяжелых JS-фреймворков.
- Все страницы являются **стандартными статическими HTML-файлами**.
- Стилизация осуществляется через **Tailwind CSS (CDN)**.
- Интерактивность обеспечивается через легкий модульный **Vanilla JavaScript** без внешних зависимостей.

## 🤖 Для ИИ-ассистентов (AI Context)

Если вы ИИ-ассистент, читающий этот репозиторий, обратите внимание на следующие правила (подробнее в [`.cursorrules`](./.cursorrules)):

1. **Никакого Node.js:** В проекте нет и не должно быть `package.json`, `npm`, `Webpack`, `Vite` и прочих сборщиков.
2. **Как редактировать HTML:** Вносите правки **только** в `index.html` и `accessible.html`. **Никогда** не редактируйте напрямую файлы с суффиксами `-en.html` или `-tt.html` — они генерируются автоматически!
3. **Как обновить языки:** После правки `index.html` или `accessible.html` вы обязаны запустить `python3 build.py`.

Система документации:
- [ARCHITECTURE.md](docs/ARCHITECTURE.md) — подробности кодовой базы и файловой структуры.
- [LOCALIZATION.md](docs/LOCALIZATION.md) — инструкция по работе с текстами и скриптом генерации `build.py`.

## Архитектура проекта

Короткая сводка структуры:

- `index.html` — Исходный файл русской версии главной страницы.
- `accessible.html` — Исходный файл версии для слабовидящих.
- `build.py` — Python-скрипт статического генератора (заменил JS-рендеринг на клиенте).
- Файлы `*-en.html` и `*-tt.html` (например, `index-en.html`) — **автогенерируемые** файлы. Не трогать руками.

Более детально стек описан в [ARCHITECTURE.md](docs/ARCHITECTURE.md).

## Мультиязычность (i18n)

Сайт доступен на трёх языках:
- 🇷🇺 **Русский (RU)** — основной
- 🏴 **Татарский (TT)** 
- 🇬🇧 **Английский (EN)**

*Ранее переводы работали через клиентский JS. Теперь они статичные.*

**Как добавить/изменить текст:**
1. Откройте `js/i18n/translations.js`.
2. Добавьте ваш текст в словари `ru`, `tt` и `en` (например: `hero_title: "Новый заголовок"`).
3. В файле `index.html` пропишите `data-i18n="hero_title"` нужному тегу.
4. В терминале запустите `python3 build.py`. Скрипт соберет статические HTML в корень.

Подробная инструкция в [LOCALIZATION.md](LOCALIZATION.md).

## Развертывание (Деплой)

Поскольку проект состоит исключительно из `.html`, `.css`, `.js` (и картинок), развертывание тривиально:

1. Выполните проверку, запустив Python-генератор: `python3 build.py`.
2. Загрузите всё содержимое папки на любой статический хостинг.
3. Поддерживается: **GitHub Pages**, **Netlify**, **Cloudflare Pages**, любой Nginx/Apache сервер, обычный FTP.

Серверная логика или билд-степы (через CI/CD) не требуются (только если вы не захотите автоматизировать запуск `build.py` при пуше).
