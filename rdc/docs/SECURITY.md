## Сообщение об уязвимостях

Мы серьезно относимся к безопасности нашего проекта. Если вы обнаружили уязвимость, пожалуйста, сообщите нам ответственно.

### Как сообщить

1. **НЕ** создавайте публичный issue для уязвимостей безопасности
2. Отправьте письмо на: **security@rdc-rt.ru**
3. Включите в сообщение:
   - Описание уязвимости
   - Шаги для воспроизведения
   - Потенциальное влияние
   - Возможные решения (если есть)

### Что ожидать

- Подтверждение получения в течение 48 часов
- Оценка уязвимости в течение 7 дней
- Регулярные обновления о статусе
- Благодарность в CHANGELOG (если желаете)

## Практики безопасности

### Защита данных

- Все персональные данные обрабатываются в соответствии с ФЗ-152
- Передача данных осуществляется по защищенным каналам (HTTPS)
- Применяется анонимизация данных пациентов

### Безопасность веб-приложения

- Регулярное обновление зависимостей
- Использование Content Security Policy (CSP)
- Защита от XSS и CSRF атак
- Валидация всех входных данных

### Рекомендации для разработчиков

1. Никогда не коммитьте секреты в репозиторий
2. Используйте `.gitignore` для конфиденциальных файлов
3. Проверяйте зависимости на известные уязвимости
4. Следуйте принципу минимальных привилегий
