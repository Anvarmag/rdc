# Архитектура проекта "РДЦ" (Радиологический дата-центр)

Проект использует архитектуру **статического сайта без процесса сборки (build pipeline) на основе Node.js**. Это осознанный выбор для ускорения разработки, упрощения деплоя и минимизации зависимостей.

## 1. Технологический стек

- **HTML5**: Семантическая структура страниц.
- **Tailwind CSS (через CDN)**: Фреймворк для стилизации элементов прямо в HTML (утилитарные классы).
- **Vanilla JavaScript**: Логика модальных окон, шапки, базовых виджетов (без React/Vue/Angular).
- **Lucide Icons**: SVG-иконки, подключаются как скрипт.
- **Python-генератор (build.py)**: Кастомное решение для статической генерации мультиязычных страниц (вместо клиентского рендеринга текстов).

## 2. Организация файловой системы (SSG Архитектура)

Проект использует классическую структуру генераторов статических сайтов:

```text
rdc-website/
├── build.py                 # Скрипт статической сборки (Генератор из src/ в dist/)
├── src/                     # 1. ПАПКА ИСХОДНИКОВ (Где идет разработка)
│   ├── index.html           # Исходник главной страницы (только RU)
│   ├── accessible.html      # Исходник версии для слабовидящих (только RU)
│   ├── css/                 # Модульные стили
│   ├── js/                  # Скрипты
│   │   ├── modules/         # Инкапсулированная UI логика
│   │   ├── data/            # Данные контента
│   │   ├── i18n/            # Словари переводов (translations.js)
│   │   └── main.js          # Инициализация JS
│   ├── images/              # Медиа и заглушки
│   ├── fonts/               # Шрифты
│   └── documents/           # PDF-документы (Лицензии)
├── dist/                    # 2. ПАПКА БИЛДА (Содержимое для сервера хостинга)
│   ├── index.html           # Сгенерированная RU версия 
│   ├── index-en.html        # Сгенерированная EN версия
│   ├── index-tt.html        # Сгенерированная TT версия
│   ├── accessible*.html     # Сгенерированные слабовидящие версии
│   └── [css, js, images...] # Ассеты (скопированы из src)
└── docs/                    # 3. ДОКУМЕНТАЦИЯ (Архитектура, Инструкции)
```

## 3. Сборка (Generation Pipeline)

Единственный инструмент сборки в проекте — скрипт `build.py`. 
Проект **не нуждается** в `npm install` или `npm run build`.

Вся архитектура отталкивается от концепции **"Master HTML"**:
- Файлы `index.html` и `accessible.html` выступают исходниками.
- В них проставлены кастомные дата-атрибуты для перевода (например `data-i18n="title"`).
- Скрипт парсит `js/i18n/translations.js`, берет словари `en` и `tt`.
- Подставляет переводы в HTML-заглушки и сохраняет как готовые к деплою файлы: `index-en.html`, `index-tt.html`, `accessible-en.html`, `accessible-tt.html`.

## 4. Разграничение ответственности

- **Tailwind**: Отвечает за 95% дизайна.
- **CSS-файлы**: Используются только для сложных анимаций (Float), размытия (backdrop-filter) и градиентного текста (text-clip), которые громоздки в классах Tailwind.
- **JS-модули**: Работают по принципу прогрессивного улучшения (Progressive Enhancement). Они оживляют существующий DOM, а не генерируют HTML.
- **Версия для слабовидящих (`accessible.html`)**: Является Полностью независимым документом, отдельным от основной темы сайта `index.html`. Изменения структуры в одной версии **не влияют** автоматически на другую, правки нужно синхронизировать вручную (или переопределять стилями).
