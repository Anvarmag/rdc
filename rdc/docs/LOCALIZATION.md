# Руководство по локализации сайта (Словарь и build.py)

В этом документе описано, как устроен процесс перевода текстов на сайте РДЦ. Сайт полностью сгенерирован в статику для максимальной производительности. Ранее переводы работали "на лету" с помощью JS, теперь используется **генератор статики на Python**.

## 1. Где хранятся переводы?

Единственный источник правды для всех текстов сайта — это файл:
`src/js/i18n/translations.js`

Структура файла представляет собой три вложенных словаря (объекта): `ru`, `en`, `tt`.
```javascript
const TRANSLATIONS = {
  // 1. Русский
  ru: {
    hero_title: "Радиологический дата-центр",
    hero_btn: "Выбрать услугу"
  },
  
  // 2. Английский
  en: {
    hero_title: "Radiological Data Center",
    hero_btn: "Choose service"
  },
  
  // 3. Татарский
  tt: {
    hero_title: "Радиология мәгълүмат үзәге",
    hero_btn: "Хезмәтне сайлау"
  }
};
```

## 2. Как связать HTML и словарь?

Для того чтобы слово на сайте подхватывало перевод из `translations.js`, нужно:
1. Обернуть текст в отдельный тег (например `<span>`, `<p>`, `<h1>`) в `src/index.html`.
2. Добавить тегу атрибут `data-i18n="ваш_КЛЮЧ"`.

*Пример:*
```html
<h1 data-i18n="hero_title">Радиологический дата-центр</h1>
```

## 3. Процесс генерации языковых версий

Если вы добавили новый ключ в `translations.js` и прописали `data-i18n` в `src/index.html` или `src/accessible.html`, эти изменения **не появятся сами по себе** в рабочей версии сайта. 

Их нужно туда сгенерировать в папку `dist/`:

1. Откройте терминал в корневой папке с сайтом.
2. Убедитесь, что установлен Python 3.
3. Выполните команду:
   ```bash
   python3 build.py
   ```
4. В консоли появится отчет о генерации:
   ```
   Processing index.html...
   Processing accessible.html...
   Done!
   ```

Скрипт `build.py` прочтет `src/index.html`, найдет все теги с `data-i18n`, подставит текст из `translations.js` (подставит "Radiological Data Center" из словаря `en`), и сохранит результат в готовый файл `dist/index-en.html`. 
Затем он сделает то же самое для татарского словаря и скопирует все картинки и стили в папку `dist/`.

#### Важно:
Никогда не редактируйте напрямую файлы в папке `dist/`. Любые внесенные туда правки будут стёрты при следующем запуске `build.py`. Редактируйте только мастер-страницы (`src/index.html`, `src/accessible.html`)!
